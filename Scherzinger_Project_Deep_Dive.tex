\section{What Is This Project?}

Scherzinger GmbH is an industrial pump manufacturer. They have 4 years (2022--2025) of historical invoice and quotation data. Before this platform, all margin analysis was manual---spreadsheets, no forecasting, no risk awareness, no way to see which customers or products were silently losing them money.

We built a \textbf{Margin Intelligence Platform} that:

\begin{itemize}
  \item \textbf{Phase 1:} Ingests their raw Excel data, cleans it, loads it into a PostgreSQL database, and provides 19 API endpoints for margin analysis, quote performance, and data quality reporting.
  \item \textbf{Phase 2:} Adds predictive intelligence---4 forecasting models, a 5-component customer risk scoring system, Monte Carlo margin simulations, seasonal pattern detection, cost trend tracking, commodity benchmarking, and walk-forward backtest validation.
  \item \textbf{Phase 3:} Builds a React frontend dashboard with 9 pages that visualize all of Phase 1 and 2's analytics---KPI cards, trend charts, customer/product tables, forecast comparison views, risk tier breakdowns, and more.
\end{itemize}

\subsection{Tech Stack}
\begin{itemize}[nosep]
  \item \textbf{Backend:} Python (FastAPI 0.109), PostgreSQL 15, SQLAlchemy 2.0
  \item \textbf{Frontend:} React 19.2 + Vite 7, Recharts 3.7, Tailwind CSS 4.2, Framer Motion
  \item \textbf{ML / Analytics:} NumPy, Pandas, SciPy, Statsmodels
  \item \textbf{Data Pipeline:} Openpyxl (Excel reading), PyArrow (Parquet), Alembic (DB migrations)
\end{itemize}

\section{How We Cleaned \& Prepared the Data}

\subsection{The Cleaning Pipeline (\code{clean\_data.py})}

\textbf{Step 1---Read Excel Sheets:} We read all 4 yearly sheets from both Excel files (2022, 2023, 2024, 2025) and concatenate them.

\textbf{Step 2---Column Mapping:} All German column names mapped to English. Types converted: dates parsed, numbers coerced, strings stripped.

\textbf{Step 3---Derived Fields Added:}
\begin{itemize}[nosep]
  \item \code{year}, \code{quarter}, \code{month} extracted from the date field
  \item \code{business\_unit} set to ``BU001'' (single business unit)
  \item Quote \code{db2\_margin} = \code{db2\_pct\_raw / 100.0}
  \item Quote \code{is\_won} = True if \code{status\_code == 4}, False if 5
\end{itemize}

\textbf{Step 4---Data Quality Flags:}

For invoices:
\begin{lstlisting}
dq_missing_margin  = TRUE if db2_margin is NULL     (20 records, 0.36%)
dq_negative_margin = TRUE if db2_margin < 0          (13 records, 0.23%)
dq_low_margin      = TRUE if 0 <= db2_margin < 0.10  (flagged but not excluded)
dq_any_issue       = TRUE if any of the above is TRUE
\end{lstlisting}

For quotes:
\begin{lstlisting}
dq_100pct_margin = TRUE if db2_pct_raw == 100.0  (802 records, 17.4%)
  -> These are quotes where HKvoll = 0, so DB2 = revenue, giving 100% "margin"
  -> They are EXCLUDED from all model training
dq_missing_cost  = TRUE if hkvoll is NULL or 0    (96 records)
dq_any_issue     = TRUE if any of the above is TRUE
\end{lstlisting}

\textbf{Step 5---Rejection Code Normalization:}
\begin{itemize}[nosep]
  \item Mixed case codes normalized to uppercase (``ka'' $\to$ ``KA'', ``Pa'' $\to$ ``PA'')
  \item Short codes mapped: ``P'' $\to$ ``PR'', ``T'' $\to$ ``TE''
  \item Reliability flag: \code{rejection\_code\_reliable = TRUE} only for 2025 data
\end{itemize}

\textbf{Step 6---Deduplication:} Quotes deduplicated on (\code{quote\_id}, \code{position}), keeping the first occurrence.

\textbf{Step 7---Output:} Clean data written to Parquet files:
\begin{itemize}[nosep]
  \item \code{invoices\_clean.parquet} (5{,}565 records)
  \item \code{quotes\_clean.parquet} (4{,}539 records after dedup)
  \item \code{customers.parquet} (967 unique)
  \item \code{products.parquet} (1{,}223 unique)
\end{itemize}

\subsection{Data Loading (\code{load\_data.py})}

Parquet files loaded into PostgreSQL via batch inserts (1{,}000 records per batch). Tables created via SQLAlchemy ORM + Alembic migrations.

\subsection{Quote--Invoice Linkage (\code{link\_quotes\_invoices.py})}

This is one of the most valuable data transformations. We link won quotes to their resulting invoices using the \code{order\_id} (Auftrag) field:

\textbf{How it works:}
\begin{enumerate}[nosep]
  \item Take all won quotes that have an \code{order\_id}
  \item Find all invoices with matching \code{order\_id}
  \item Create a link record capturing: quoted margin, actual margin, the gap, and days between quote and invoice
\end{enumerate}

\textbf{Linkage Results:}
\begin{itemize}[nosep]
  \item Won quotes with \code{order\_id}: 1{,}724 (99.5\% of won quotes)
  \item Successfully matched to invoices: 1{,}406 (89.3\%)
  \item Unmatched: 169 (53 were service orders with ``S-'' prefix, 116 were recent/pending)
  \item Total link records created: 1{,}957
\end{itemize}

\textbf{Margin Gap Calculation:}
\begin{lstlisting}
margin_gap = quoted_db2_margin - actual_db2_margin
\end{lstlisting}

\begin{itemize}[nosep]
  \item \textbf{Positive gap} = we quoted higher than what we actually invoiced (margin erosion)
  \item \textbf{Negative gap} = actual margin was better than quoted (favorable)
\end{itemize}

\textbf{Key Findings:}
\begin{itemize}[nosep]
  \item Mean margin gap: \textbf{+1.9\pp{}} (we consistently quote slightly higher than we deliver)
  \item Gap by year: 2022: 2.3\pp{}, 2023: 1.9\pp{}, 2024: 2.1\pp{}, 2025: 1.4\pp{} (improving!)
  \item Average days from quote to invoice: \textbf{$\sim$62 days} (median $\sim$46 days)
\end{itemize}

\section{Phase 2---Forecasting, Risk \& ML Models}

Phase 2 adds \textbf{predictive intelligence} via 9 additional API endpoints and 7 new database tables.

\subsection{Margin Forecasting (4 Models)}

We forecast DB2 margin forward at 1, 3, 6, and 12 month horizons for: Top 200 customers (by revenue), Top 300 products (by revenue), and all 9 commodity groups.

\subsubsection{Model 1: Exponential Moving Average (EMA)}
\begin{itemize}[nosep]
  \item Requires: $\geq$12 months of data
  \item How: Applies exponential weighted mean with span=6, giving more weight to recent months
  \item Confidence interval: $\pm 1.645 \times \sigma_{\text{residuals}}$ (90\% CI)
  \item Good for: Short-term smoothing, reacting to recent changes
\end{itemize}

\subsubsection{Model 2: Linear Trend}
\begin{itemize}[nosep]
  \item Requires: $\geq$18 months of data
  \item How: Fits a straight-line regression (OLS) on monthly margin values over time
  \item Formula: $\hat{m} = \beta_0 + \beta_1 \times t$
  \item Reports $R^2$ (goodness of fit)
  \item Good for: Detecting directional trends (is margin going up or down?)
\end{itemize}

\subsubsection{Model 3: Seasonal Decomposition}
\begin{itemize}[nosep]
  \item Requires: $\geq$24 months of data
  \item How: Uses statsmodels additive seasonal decomposition with 12-month period
  \item Separates margin into: trend + seasonal + residual
  \item Prediction = extrapolated trend + seasonal factor for target month
  \item Good for: Capturing repeating monthly patterns
\end{itemize}

\subsubsection{Model 4: Ensemble (Weighted Average)}
\begin{itemize}[nosep]
  \item Requires: $\geq$18 months of data
  \item If seasonal available: \textbf{30\% EMA + 30\% Linear + 40\% Seasonal}
  \item If seasonal unavailable: \textbf{50\% EMA + 50\% Linear}
  \item Confidence interval: $[\min(\text{all lowers}),\; \max(\text{all uppers})]$
  \item Good for: Robust predictions that balance multiple perspectives
\end{itemize}

\textbf{All predictions clipped to $[-1.0, 1.0]$ range.}

\subsection{Monte Carlo Simulations}

We run stochastic simulations to model the probability distribution of future margins.

\textbf{How it works (step by step):}
\begin{enumerate}[nosep]
  \item Take a customer/product/group's monthly margin history
  \item Compute monthly returns: $r_t = m_t - m_{t-1}$
  \item Calculate drift ($\mu = \bar{r}$) and volatility ($\sigma = \text{std}(r)$)
  \item Simulate 10{,}000 paths (5{,}000 for customers): each month $m_{t+1} = \text{clip}(m_t + \mathcal{N}(\mu, \sigma),\; -1,\; 1)$
  \item From 10{,}000 final margins, compute percentiles ($p_5, p_{25}, p_{50}, p_{75}, p_{95}$)
  \item Compute $\Pr(\text{margin} < 50\%)$
\end{enumerate}

\section{All Formulas \& Calculations}

\subsection{Core Margin Formulas (German Accounting)}

\begin{align}
\text{DB I} &= \text{Revenue} - (\text{HKvar\_per\_unit} \times \text{Quantity}) \label{eq:db1} \\
\text{DB II} &= \text{Revenue} - (\text{HKvoll\_per\_unit} \times \text{Quantity}) \label{eq:db2} \\
\text{DB I Margin} &= \frac{\text{DB I Total}}{\text{Revenue}} \\
\text{DB II Margin} &= \frac{\text{DB II Total}}{\text{Revenue}} \quad \leftarrow \text{THE KEY METRIC}
\end{align}

\subsection{Cost Component Breakdown}

\begin{align}
\text{Material \%} &= \frac{\text{material\_per\_unit}}{\text{hkvoll\_per\_unit}} \times 100 \\
\text{Direct Mfg \%} &= \frac{\text{fek\_per\_unit}}{\text{hkvoll\_per\_unit}} \times 100 \\
\text{Outsourcing \%} &= \frac{\text{fv\_per\_unit}}{\text{hkvoll\_per\_unit}} \times 100
\end{align}

\subsection{Quote--Invoice Margin Gap}

\begin{align}
\text{margin\_gap} &= \text{quoted\_db2\_margin} - \text{actual\_db2\_margin} \\
\text{days\_to\_invoice} &= \text{invoice\_date} - \text{quote\_date} \;\text{(in days)}
\end{align}

\subsection{Revenue-Weighted Margin}

\[
\text{weighted\_margin} = \frac{\sum \text{db2\_total}}{\sum \text{revenue}}
\]

This is more accurate than simple averaging because a \EUR500K invoice at 70\% counts much more than a \EUR500 invoice at 90\%.

\subsection{Win Rate}

\[
\text{win\_rate} = \frac{\text{COUNT}(\text{is\_won} = \text{TRUE})}{\text{COUNT}(\text{all quotes})}
\]

\subsection{Price Sensitivity T-Test}

\begin{lstlisting}
Group A: margins of all WON quotes (excluding dq_100pct_margin)
Group B: margins of all LOST quotes (excluding dq_100pct_margin)
Test: Independent samples t-test (scipy.stats.ttest_ind)
Result: p-value = 0.006 (significant at alpha=0.05)
Interpretation: Lost quotes had higher margins -> pricing affects win probability
\end{lstlisting}

\subsection{EMA Forecast Formula}

\begin{lstlisting}[language=Python]
ema = series.ewm(span=6, adjust=False).mean()
predicted = ema.iloc[-1]  # last EMA value projected forward
residuals = series - ema_fitted
ci_width = 1.645 * residuals.std()  # 90% confidence
lower = predicted - ci_width
upper = predicted + ci_width
\end{lstlisting}

\subsection{Linear Trend Forecast Formula}

\begin{lstlisting}[language=Python]
from scipy.stats import linregress
x = range(len(series))
slope, intercept, r_value, p_value, std_err = linregress(x, series)
r_squared = r_value ** 2
predicted = intercept + slope * (len(series) + months_ahead - 1)
ci_width = 1.645 * std(residuals)
\end{lstlisting}

\subsection{Seasonal Decomposition Forecast}

\begin{lstlisting}[language=Python]
from statsmodels.tsa.seasonal import seasonal_decompose
result = seasonal_decompose(series, model='additive', period=12)
# Extrapolate trend via linear regression on trend component
# Add seasonal factor for the target month

\end{lstlisting}

\subsection{Ensemble Forecast}

\begin{lstlisting}[language=Python]
# With seasonal (requires >= 24 months):
weights = [0.3, 0.3, 0.4]  # EMA, Linear, Seasonal
predicted = 0.3 * ema_pred + 0.3 * linear_pred + 0.4 * seasonal_pred

\end{lstlisting}

\subsection{Risk Score Components}

\begin{lstlisting}[language=Python]
# Component 1: Margin Trend (30%)
delta = mean(last 4 quarters) - mean(first 4 quarters)
margin_trend_score = clip((0.05 - delta) / 0.15, 0, 1)

# Component 2: Margin Gap (25%)
gap_score = clip(avg_gap / 0.15, 0, 1)

# Component 3: Volume Trend (20%)
change = (recent_revenue - prior_avg) / prior_avg
volume_score = clip((0.10 - change) / 0.40, 0, 1)

# Component 4: Win Rate (15%)
winrate_score = clip((0.50 - recent_win_rate) / 0.35, 0, 1)

# Component 5: Rejection Pattern (10%)
price_ratio = count(PA + PR) / count(all lost)
rejection_score = clip((price_ratio - 0.10) / 0.40, 0, 1)

# Composite Score
composite = 0.30*margin_trend + 0.25*gap + 0.20*volume
          + 0.15*winrate + 0.10*rejection
\end{lstlisting}

The composite score formula in mathematical notation:

\[
S = 0.30\,S_{\text{margin}} + 0.25\,S_{\text{gap}} + 0.20\,S_{\text{volume}} + 0.15\,S_{\text{winrate}} + 0.10\,S_{\text{rejection}}
\]

\subsection{Monte Carlo Simulation}

\begin{lstlisting}[language=Python]
returns = np.diff(margin_series)  # monthly changes
mu = returns.mean()               # drift
sigma = returns.std()             # volatility
np.random.seed(42)

for sim in range(10000):
    margin = last_known_margin
    for month in range(horizon):
        shock = np.random.normal(mu, sigma)
        margin = clip(margin + shock, -1.0, 1.0)
    final_margins.append(margin)

# Results
p5  = np.percentile(final_margins, 5)
p25 = np.percentile(final_margins, 25)
p50 = np.median(final_margins)
p75 = np.percentile(final_margins, 75)
p95 = np.percentile(final_margins, 95)
prob_below_threshold = np.mean(final_margins < 0.50)
\end{lstlisting}

\subsection{Seasonal Index}

\[
I_m = \frac{\bar{m}_{\text{month}}}{\bar{m}_{\text{grand}}}
\]

If sample count $<$ minimum threshold, index defaults to 1.0.

\subsection{Cost Change Percentage}

\[
\bar{C} = \frac{\sum (\text{hkvoll\_per\_unit} \times \text{quantity})}{\sum \text{quantity}}
\qquad
\Delta_{\text{cost}} = \frac{\bar{C}_{\text{current}} - \bar{C}_{\text{prior}}}{|\bar{C}_{\text{prior}}|}
\]

Warning flag if $|\Delta_{\text{cost}}| > 0.50$ (50\% swing).

\subsection{Backtest Metrics}

\begin{align}
\text{MAE}  &= \frac{1}{n}\sum_{i=1}^{n} |y_i - \hat{y}_i| \\
\text{RMSE} &= \sqrt{\frac{1}{n}\sum_{i=1}^{n} (y_i - \hat{y}_i)^2} \\
\text{MAPE} &= \frac{1}{n}\sum_{i=1}^{n} \frac{|y_i - \hat{y}_i|}{|y_i|} \\
\text{Dir.\ Acc.} &= \frac{1}{n}\sum_{i=1}^{n} \mathbf{1}\!\left[\text{sign}(\Delta \hat{y}_i) = \text{sign}(\Delta y_i)\right]
\end{align}

\section{Future Phases---What They Can Expect}

\subsection{Phase 4: Pricing \& FX Analysis (Planned)}

\begin{enumerate}
  \item \textbf{Price Optimization Engine:} Analyzes the relationship between price level and win probability. Recommends optimal price points that maximize expected profit (margin $\times$ win probability). Uses the price sensitivity data we already have ($p = 0.006$).

  \item \textbf{FX Impact Analysis:} Models how currency movements affect landed cost and margin. Provides hedging recommendations when FX exposure exceeds thresholds.

  \item \textbf{Price Governance Framework:} Automated approval workflows: discounts $>$5\% require manager approval, $>$10\% require director. Tracks discount frequency by sales rep, customer, product.

  \item \textbf{What-If Simulator:}
  \begin{itemize}[nosep]
    \item ``What happens to margin if material costs rise 10\%?''
    \item ``What if we raise prices on BKAGG products by 5\%?''
    \item ``How much volume can we afford to lose if we increase price by 3\%?''
  \end{itemize}
\end{enumerate}

\subsection{Phase 5: Advanced ML Analytics (Planned)}

\begin{enumerate}
  \item \textbf{ML Churn Prediction:} Trained classifier (Random Forest or XGBoost) predicting 12-month churn probability. Features: order frequency, margin trend, volume trend, days since last order, quote win rate. Expected accuracy: 78--85\%.

  \item \textbf{Customer Lifetime Value (CLV) Modeling:}
  \[
    \text{CLV} = \sum_{t=1}^{T} \text{predicted\_revenue}_t \times \text{predicted\_margin}_t \times \text{retention\_prob}^t
  \]

  \item \textbf{Demand Classification:} Segments each SKU into demand patterns: Steady, Seasonal, Lumpy, Intermittent, New. Uses coefficient of variation and order frequency analysis.

  \item \textbf{Advanced Anomaly Detection:} ML-based outlier detection (Isolation Forest) on margin, revenue, and cost patterns. Automatic alerts when metrics deviate from expected patterns.
\end{enumerate}

\subsection{Phase 6+ (Potential Future)}

  \item \textbf{Email/Alert System:} Automated notifications on risk score changes

  \item \textbf{Supplier Cost Intelligence:} Track supplier price changes
