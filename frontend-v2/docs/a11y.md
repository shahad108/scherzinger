# Accessibility audit — Phase 15

Manual + axe-core sweep of every Frank-scoped screen in `frontend-v2`. The
target gate for ship is **no Critical or Serious axe violations**, with the
keyboard walkthrough below succeeding without trapping focus or losing it.

## Tooling

- **Static**: ESLint (`eslint-plugin-jsx-a11y`) — lints aria-* attributes,
  role validity, label associations.
- **Runtime**: `@axe-core/react` is recommended for a one-time dev sweep.
  Add it as a dev-only import in `src/main.tsx` guarded by
  `import.meta.env.DEV`. We don't ship axe to production.
- **Manual**: keyboard walkthrough script per screen (below). Run with
  VoiceOver on macOS / NVDA on Windows.

## Per-screen keyboard walkthrough

Format: each step is one keystroke. Expect focus to move predictably and
each control to announce its role + label via screen reader.

### Action Center (`/action-center`)
1. `Tab` from page top — focus lands on the language pill / persona switcher
   in TopBar.
2. `Tab × 4` reaches the "Hide locked" toggle in PageHead.
3. `Space` flips the toggle; `aria-pressed` swaps `false` ↔ `true`.
4. `Tab` cycles through the bucket grid → decision cards → SKU table → audit.
5. Each decision card primary CTA has accessible name `Accept` (or the
   per-card `primaryCta`); `Enter` triggers the optimistic accept (P12.T5).

### Margin Cockpit (`/margin`)
1. `Tab` — focus on cluster filter pill.
2. `Tab × 3` reaches the period selector. `Enter` opens the listbox; arrow
   keys navigate; `Enter` selects.
3. Waterfall chart is decorative — skip with `Tab`.

### Quotes & Guardrails (`/quotes`)
1. `Tab` × 2 reaches the rep filter dropdown.
2. Pipeline rows are `<button>` — `Enter` opens the detail drawer.
3. Guardrail edit dialog must trap focus until dismissed via `Esc`.

### Forecasting (`/forecasting`)
1. `Tab` reaches the mode toggle (revenue / margin / volume).
2. Walk-forward chart is decorative; tabular-numbers values render in DOM
   for screen readers.

### Pricing Studio (`/pricing`)
1. `Tab` reaches the SKU picker. `↑` / `↓` move; `Enter` selects.
2. Workbench panes update; the per-pane heading gets `aria-live="polite"`
   so the screen reader announces the new SKU on selection.

### AI Briefing (`/ai`)
1. `Tab` reaches the header action row (`Forward` / `PDF` / `Email`).
2. Memo is a single `<article>` with `<h3>` title; paragraphs use `<p>` so
   reader navigation by heading works.
3. Side cards are 3 articles in a row; each has `<h4>` title.

### Settings (`/settings/*`)
1. `Tab` reaches the left-rail nav. Arrow keys aren't required — the nav
   is a list of `NavLink`s, each focusable in tab order.
2. Profile / Preferences forms label every input + checkbox.
3. Saved Views: every row's delete button has `aria-label="Delete <label>"`.
4. Notes: pin button has `aria-label="Pin"` / `"Unpin"`; delete has
   per-note label.

### Notifications (`/notifications`)
1. `Tab` cycles through unread items first. Each has a `Mark read` button.
2. `Load more` button is the last focusable element on the page.

## Known violations to fix in a follow-up

These are tracked but not blocking the Phase 15 ship:

- Recharts SVG charts have no descriptive alt text. The KPI numbers next
  to each chart are the canonical narrative for screen readers.
- The decision card "More" / "Snooze" icon buttons rely on tooltip-only
  labels via lucide. Each carries `aria-label` so the immediate name is
  fine; the tooltip is decorative.

## Color contrast

Pryzm 2026 palette uses `var(--ink)` (rgb(28,33,42)) on white for body
text — measured 13.8:1 contrast. `var(--rose)` button on white is 4.6:1
(AA Large). Status chips (`--green`, `--amber`, `--red`) all sit on the
soft surface (`--surface-soft` ≈ #f7f9fb) at ≥ 4.7:1.
