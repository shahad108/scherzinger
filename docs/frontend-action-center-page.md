## 1. Entry point

| File | Role |
|---|---|
| `frontend-v2/src/features/action-center/index.tsx` | Page shell. Fetches the composed payload via `useActionCenter`, owns the two page-level toggles (`hideLocked`, `showAll`) and the `trustTile` drawer state, then walks the payload top-to-bottom rendering each block (or a `DegradedBlock` when the composer marks the slice degraded). Mounted by the router at `frontend-v2/src/app/router.tsx:85` as `/action-center` and is the post-login landing page for Frank (`router.tsx:68`, `Login.tsx:96`). |

### URL parameters (read by `ActionCenterPage`)

The page itself does **not** read any URL params — `index.tsx` never calls `useSearchParams`. All filter state is component-local React state (`hideLocked`, `showAll`, `trustTile`). Cross-screen deep links **into** Action Center append query strings (`?queue=renewals&customer=…&source=forecasting-customers`, `?tab=negotiation`, `?focus=rec-…`, `?persona=till`), but no code on this page consumes them today.

| Param | Read by | Effect |
|---|---|---|
| `queue` | — (not consumed) | Reserved for renewal-queue deep-links from `forecasting/PerCustomerTab.tsx:252` and shared notifications. Component-level handling is a follow-up. |
| `customer` | — (not consumed) | Same as above. |
| `source` | — (not consumed) | Inbound provenance tag; flows back out through the action intents created on this page (every fallback intent stamps `source: 'action-center'`). |
| `tab` | — (not consumed) | `?tab=negotiation` from `forecasting/CrossLinkStrip.tsx:35` — would target the negotiation block, but no scroll-to handler exists yet. |
| `focus` | — (not consumed) | `?focus=rec-…` is generated by the `share_decision` flow (mock store + tests at `tests/forms/form-drawers.test.tsx:200`) for "Open audit trail" deep-links; the receiving page does not yet auto-scroll. |
| `persona` | — (not consumed) | Reserved by Till's persona overview cross-link (`tests/persona-overview/persona-pages.test.tsx:48`). |

### Page-level state (`index.tsx`)

| State | Initial | Effect |
|---|---|---|
| `hideLocked` | `false` | Sent to BFF as `hide_locked` query param (`useActionCenter`); composer drops locked-contract rows when true. Toggled by `PageHead` "Hide locked / Locked hidden" pill. |
| `showAll` | `false` | Bumps the BFF `limit` from `5` → `200` for every paginated list block (decisions, SKU table, rejections). The composer still applies per-block floors (`decisions ≥ 3`, `sku_table ≥ 50`) so collapsing back never truncates below the canonical view. Toggled by `PageHead` "Show all / Showing all" pill. |
| `trustTile` | `null` | When set to a `TrustTile`, opens `TrustDrawer`. Set by `TrustStrip` row click; cleared by the drawer's close handler. |

### Data source

- Hook: `useActionCenter({ hide_locked, limit })` in `frontend-v2/src/data/api/useActionCenter.ts:44`.
- Endpoint: BFF `GET /screens/action-center`. Composer attaches per-block status (`live` / `empty` / `degraded`) plus coverage + reason metadata under `meta.blocks`.
- Cache: `staleTime: 60_000`, `refetchInterval: 60_000` (foreground only). Comment at `useActionCenter.ts:53` flags this as the Phase 14 freshness compromise — worst-case staleness is ~120s.
- Mock-mode parity: `filterByLocalProposals` (`useActionCenter.ts:20`) reads `sessionStorage[pryzm_v2_synth_proposals]` and hides decisions whose `recommendationId` already has a synthetic proposal, mirroring the live `workflow_service.get_recommendation_status_map` filter so demos see "accept → refresh → card gone" without a server.
- Returns `ActionCenterData` (see `frontend-v2/src/types/index.ts:331`).

### Page-shell render order (top → bottom)

```
PageHead                                  (always)
DataFreshnessStrip                        (when data.meta.dataFreshness)
[ DegradedBlock | <CoverageBadge> + MovableHero ]
[ DegradedBlock | BucketGrid ]
[ DegradedBlock | DecisionCards ]
[ DegradedBlock | <CoverageBadge> + TrustStrip ]
TrustDrawer                               (portal — opened by TrustStrip)
[ DegradedBlock | <CoverageBadge> + LostQuoteCard ]
[ DegradedBlock | SkuTable ]
[ DegradedBlock | LongTailCoverage ]
[ DegradedBlock | NegotiationCockpit ]
[ DegradedBlock | AbTestList ]
[ DegradedBlock | RejectionList ]
[ DegradedBlock | AuditTrail ]
ReportCard                                (always; disabled when audit block is degraded)
```

Every block branches on `data.meta?.blocks.<id>.status === 'degraded'` and falls back to a `DegradedBlock` with `title` / `hint` / `traceId`. Three blocks (`movableHero`, `trust`, `lostQuote`) also render an inline `CoverageBadge` from `meta.blocks.<id>.coverage` above the headline.

Loading state: `ActionCenterSkeleton` (single page-level shimmer). Error state: red inline `"Fehler: …"` line, no skeleton.

## 3. Page-level fallback intents

The comment at `index.tsx:26` flags these as defensive only — production composer payloads always attach `action`.

## 5. Action dispatcher & mutations

| Hook | Source file | Notes |
|---|---|---|
| `useUiAction()` | `frontend-v2/src/hooks/useUiAction.ts` | Single page-wide dispatcher. Delegates to `executeUiAction` in `lib/uiActions.ts`, threading `navigate` (React Router), `toast`, and `drawer` from `actionFeedbackStore`. Errors caught and toasted as `error`. |
| `useAcceptDecision` | `frontend-v2/src/data/api/useActions.ts:247` | Wraps `runAction('accept_recommendation', …)`. Invalidates `qk.actionCenter()`, `qk.auditTrail('30d')`, `qk.shell()`. |
| `useDeclineDecision` | `useActions.ts:249` | `decline_recommendation`. Same invalidation set. |
| `usePartialAccept` | `useActions.ts:251` | `partial_accept`. Submitted from the form drawer, not directly from this page. |
| `useStartAbTest` / `useStopAbTest` | `useActions.ts:254`/`256` | A/B lifecycle. Invalidates `qk.actionCenter()` + `qk.studio()`. |
| `useGenerateActionCenterReport` / `useSendReport` | `data/api/useReportJob.ts:106`/`117` | Report lifecycle for `ReportCard`. |
| `useTrustDrawer` | `data/api/useTrustDrawer.ts` | Lazy — only fires when the drawer opens (`enabled` flag from `TrustDrawer` `open` prop). 404 / network failures fall back to `EMPTY` so the drawer renders an honest empty state. |

All mutations POST through `runAction` (`useActions.ts:152`) with an idempotency header `x-pryzm-idempotency-key: <kind>:<target_id>` so retries from React Query never double-write to the audit log. In mock mode, `syncSynthProposalForMock` mirrors accept/partial-accept results into `sessionStorage[pryzm_v2_synth_proposals]` so the next `useActionCenter` refetch hides the accepted card (`useActionCenter.ts:20`).

## 8. Open follow-ups (documented inline)

- **Per-block skeletons** — `ActionCenterSkeleton.tsx:4` flags Phase 14 / streaming-SSR rewrite. Today the page renders one atomic shimmer skeleton until every block resolves.
- **60s auto-refresh tradeoff** — `useActionCenter.ts:52` comment owns the ~120s worst-case freshness window inherited from the BFF's 60s server cache. Acceptable for analyst workflows; tighter polling would need server-side cache invalidation.
- **URL-param consumers** — every inbound deep-link query (`?queue=`, `?customer=`, `?focus=rec-…`, `?tab=negotiation`, `?persona=`) is currently written but not read. No `useSearchParams` on the page yet; scroll-to-block and pre-filter behaviors are pending.
- **`?tab=negotiation`** from `forecasting/CrossLinkStrip.tsx:35` lands the user on the page but does not scroll to or expand `NegotiationCockpit`.
- **Workspace-scope / Export drawer items** are static read-only — no live data behind them (`PageHead.tsx:151-181`).
- **Mock-mode synthetic stores** (`pryzm_v2_synth_proposals`, `pryzm_v2_synth_reports`) need to be cleared between sessions or they leak across demo personas; no UI hook to reset them today.
- **No Playwright coverage** for this page yet — all E2E specs are forecasting-only.
